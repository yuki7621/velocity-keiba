# velocity-keiba 仕様書

## 1. システム概要

**velocity-keiba** は、JRA（日本中央競馬会）のレースデータをAIで分析し、
回収率の高い馬を自動で見つけ出す競馬予測システムです。

```
【全体の流れ】

  netkeiba.com → スクレイピング → データベース → AI分析 → 予測結果
  （レースデータ）  （自動収集）     （SQLite）    （LightGBM）  （Webアプリで表示）
```

### 4.3 予測戦略

このシステムは「当てる」のではなく「儲かる馬を探す」戦略を取ります。

```
【従来の予測】                【このシステムの予測】
  AIが「勝ちそう」と            AIが「市場に過小評価
  思った馬に賭ける              されている」馬に賭ける
       ↓                           ↓
  人気馬ばかり当たる            穴馬を効率よく拾う
       ↓                           ↓
  的中率は高いが                的中率は低いが
  配当が低い                    配当が大きい
       ↓                           ↓
  回収率 75%（赤字）            回収率 200%超（黒字）
```

#### バリューベット戦略の仕組み

```
  エッジ = AI予測確率 − 市場の暗黙確率

  例: 単勝オッズ20倍の馬
      市場の評価   = 3 / 20 = 15%（複勝的中率）
      AIの予測     = 35%
      エッジ       = 35% − 15% = +20%
      → エッジが大きい = 市場が過小評価 = 買い！
```

### 4.4 バックテスト結果（v3モデル）

テスト期間: 2025年1月〜12月

| エッジ閾値 | 賭け数 | 的中率 | 概算回収率 |
|-----------|--------|--------|-----------|
| 0.05 | 8,968 | 26.9% | 160.3% |
| 0.10 | 5,381 | 33.4% | 182.8% |
| 0.15 | 3,409 | 41.2% | 213.0% |
| 0.20 | 2,266 | 50.1% | 246.9% |

※ 概算回収率は「複勝配当 ≒ 単勝オッズ × 0.3」で計算した参考値です。

## 5. Webアプリ画面仕様

### 5.1 画面一覧

| 画面名 | 機能 |
|--------|------|
| 予測 | 日付・競馬場・モデルを選択して予測を実行 |
| 馬場傾向 | 指定日の内外バイアス・脚質バイアスをグラフ表示 |
| データ更新 | 月指定で netkeiba からデータをスクレイピング |
| バックテスト | 戦略・パラメータを調整して回収率をシミュレーション |
| DB状況 | データ件数、期間、年別・競馬場別の統計を表示 |

### 5.2 予測画面の出力

各レースの出走馬について以下を表示:

| 表示項目 | 説明 |
|----------|------|
| 馬番 | 出走番号 |
| 馬名 | 馬の名前 |
| AI確率 | AIが予測した複勝確率 |
| オッズ | 単勝オッズ |
| 期待値 | AI確率 × 概算複勝配当 |
| エッジ | AI確率 − 市場の暗黙確率 |
| 推奨 | ★〜★★★ のランク |

#### 推奨マーク基準

| マーク | 条件 |
|--------|------|
| ★★★ | エッジ ≥ 0.15 かつ 期待値 ≥ 1.0 |
| ★★ | エッジ ≥ 0.10 かつ 期待値 ≥ 0.8 |
| ★ | エッジ ≥ 0.05 |
| （なし） | エッジ < 0.05（見送り） |
