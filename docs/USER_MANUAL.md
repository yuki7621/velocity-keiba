# velocity-keiba 取り扱い説明書

## はじめに

このアプリは、競馬のレースデータをAIが分析して
**「この馬は人気以上に走りそう」** という馬を見つけてくれるツールです。

難しい操作は必要ありません。基本的に **ボタンを押すだけ** で使えます。

## 1. 最初のセットアップ（初回のみ）

> この作業は最初の1回だけ必要です。2回目以降は不要です。

### 必要なもの

- Windows 10 または 11 のパソコン
- インターネット接続

### 手順

#### ステップ1: Pythonをインストール

1. 以下のサイトを開きます
   - https://www.python.org/downloads/
2. **「Download Python 3.xx.x」** の黄色いボタンをクリック
3. ダウンロードしたファイルを開きます
4. **⚠️ 重要**: 画面下の **「Add python.exe to PATH」にチェックを入れる**
5. **「Install Now」** をクリック
6. 「Setup was successful」と出たら完了

#### ステップ2: アプリをセットアップ

1. このプロジェクトのフォルダを開きます
2. **`setup.bat`** をダブルクリック
3. 黒い画面（コマンドプロンプト）が開いて自動でセットアップが進みます
4. 「セットアップ完了！」と表示されたら完了
5. 何かキーを押して画面を閉じます

```
⏱️ かかる時間: 約3〜5分
```

#### ステップ3: データを取り込む（受け取ったデータがある場合）

誰かから `keiba_export.tar.gz` ファイルを受け取った場合:

1. そのファイルをプロジェクトの `exports` フォルダに入れます
2. `start.bat` をダブルクリックしてアプリを起動
3. 左メニューの **「🔄 データ更新」** は不要です（データは読み込み済み）

データファイルがない場合は、アプリ起動後に自分でデータを取得できます（後述）。

## 2. アプリの起動方法

### 起動する

1. プロジェクトフォルダにある **`start.bat`** をダブルクリック
2. 黒い画面が開いて数秒待つと、自動でブラウザが開きます
3. ブラウザに競馬予想AIの画面が表示されます

```
✅ 画面が出れば成功です！
```

> ブラウザが自動で開かない場合は、手動でブラウザを開いて
> アドレス欄に `http://localhost:8501` と入力してください。

### 終了する

1. 黒い画面（コマンドプロンプト）を閉じるだけでOK
2. ブラウザのタブも閉じてください

## 3. 毎週の使い方（土曜→日曜の流れ）

```
┌─────────────────────────────────────────────────┐
│                                                 │
│   【土曜の夜】                                    │
│    アプリを起動 → 「データ更新」でその月のデータを取得  │
│    （10〜20分かかります。放置でOK）                  │
│                                                 │
│          ↓                                      │
│                                                 │
│   【日曜の朝】                                    │
│    アプリを起動 → 「予測」で明日のレースを予測         │
│    → ★マークの馬をチェック！                       │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 土曜の夜にやること（5分で完了）

1. `start.bat` でアプリを起動
2. 左メニューから **「🔄 データ更新」** を選ぶ
3. 「年」と「月」を今月に合わせる
4. **「この月のデータを取得」** ボタンを押す
5. 進行状況が表示されるので、完了まで待つ
6. 「完了！」と表示されたらOK

```
⏱️ かかる時間: 月にもよりますが 10〜30分
💡 取得中もPCは普通に使えます
```

### 日曜の朝にやること（3分で完了）

1. `start.bat` でアプリを起動
2. 左メニューから **「📊 予測」** を選ぶ
3. 「予測する日付」で **今日の日付** を選ぶ
4. 「競馬場」で **今日開催の競馬場** を選ぶ（自動で表示されます）
5. 「モデル」は **「lightgbm_v3」** のままでOK
6. **「予測を実行」** ボタンを押す
7. しばらく待つと結果が表示されます

```
⏱️ 初回は特徴量の計算に2〜3分かかります
   2回目以降はキャッシュが効くため数秒で表示されます
```

## 4. 各画面の使い方

### 📊 予測（メイン画面）

**何ができる？**: 指定した日のレースで「買い」の馬を見つけます

**操作手順**:
1. 予測する日付を選ぶ
2. 競馬場を選ぶ（複数選択可）
3. 「予測を実行」を押す

**表示される情報**:
- **馬場傾向**: 前日のバイアス（内枠/外枠、先行/差しのどちらが有利か）
- **バリューベット一覧**: 最もおすすめの馬のリスト
- **レース別予測**: 全レースの詳細予測

---

### 📈 馬場傾向

**何ができる？**: 特定の日の馬場の傾向を詳しく分析します

**操作手順**:
1. 競馬場を選ぶ
2. 日付を選ぶ
3. 「分析実行」を押す

**表示される情報**:
- **枠順バイアス**: 内枠と外枠、どちらが有利だったか
- **脚質バイアス**: 逃げ先行と差し追込、どちらが有利だったか
- **馬場スピード**: 時計が速い馬場か、タフな馬場か
- **上がり傾向**: 末脚勝負になりやすいかどうか

**見方のコツ**:
```
例: 「内枠有利 ◎」と出たら...
  → 日曜も同じ傾向が続く可能性が高い
  → 内枠の馬を重視する判断材料になる

例: 「先行有利 ◎」と出たら...
  → 前に行ける馬（逃げ・先行タイプ）を重視する
```

---

### 🔄 データ更新

**何ができる？**: netkeiba.com から最新のレース結果を取得します

**操作手順**:
- **月を指定して更新**: 年と月を選んで「この月のデータを取得」
- **今月のデータを更新**: 「今月のデータを更新」を押すだけ

**注意点**:
- 取得中はブラウザを閉じないでください
- インターネット接続が必要です
- netkeiba.com への負荷を避けるため、1レースごとに2秒間隔を空けています

---

### 🧪 バックテスト

**何ができる？**: 「もし過去にこの戦略で賭けていたら回収率はどうだったか？」をシミュレーションします

**操作手順**:
1. モデルを選ぶ（通常は「lightgbm_v3」）
2. 戦略を選ぶ:
   - **バリューベット（エッジ）**: おすすめ。AIと市場の確率差で判断
   - **期待値（EV）**: 期待リターンで判断
3. オッズ範囲を設定（初期値のままでOK）
4. 閾値を選ぶ（複数選択可）
5. 「バックテスト実行」を押す

**結果の見方**:
- **回収率100%超**: この戦略なら利益が出ていた
- **回収率100%未満**: この戦略では損をしていた
- **グラフ**: 閾値を上げると賭け数は減るが回収率は上がる傾向

---

### ℹ️ DB状況

**何ができる？**: データベースにどれくらいデータが入っているか確認します

**表示される情報**:
- レース数、出走データ数、馬数、騎手数
- データの期間
- 年別のレース数グラフ
- 競馬場別のレース数
- 最新のレース一覧

## 5. 予測結果の見方

### ★マークの意味

| マーク | 意味 | おすすめ度 |
|--------|------|-----------|
| ★★★ | AIが「かなりお買い得」と判断 | 最もおすすめ |
| ★★ | AIが「お買い得」と判断 | おすすめ |
| ★ | AIが「やや有利」と判断 | 注目 |
| （なし） | AIが「妥当な評価」と判断 | 見送り |

### 数値の見方

#### AI確率
AIが計算した「この馬が3着以内に入る確率」です。
```
  高い例: 65.2% → AIはかなり来ると予測
  低い例: 8.3%  → AIはあまり来ないと予測
```

#### オッズ
馬券売り場で表示される単勝オッズです。
```
  低い例: 2.5倍 → 人気馬（みんなが買っている）
  高い例: 45.0倍 → 人気のない馬（あまり買われていない）
```

#### 期待値
「この馬に100円賭けたら平均いくら返ってくるか」の目安です。
```
  1.00以上 → 期待リターンがプラス（買う価値あり）
  1.00未満 → 期待リターンがマイナス

  例: 期待値 1.75 → 100円賭けると平均175円返ってくる計算
```

#### エッジ
AIの評価と市場の評価の差です。プラスが大きいほどお買い得。
```
  +0.20 → AIは市場より20%も高く評価（かなりお買い得）
  +0.10 → AIは市場より10%高く評価（お買い得）
  +0.02 → ほぼ市場通りの評価（妙味なし）
  -0.05 → 市場のほうが高く評価（人気しすぎ）
```

### 具体的な使い方の例

```
日曜の朝、予測を実行して以下の結果が出たとします:

  中山11R 芝2000m
  馬番  馬名          AI確率  オッズ  期待値  エッジ   推奨
   3   ○○○○○○     38.1%   15.3   1.75   +0.184  ★★★
   7   △△△△△      29.5%   28.4   2.52   +0.144  ★★
   1   □□□□□□     52.1%    2.8   0.44   -0.021
  12   ▽▽▽▽▽      11.3%   42.0   1.43   +0.042  ★

  → 3番と7番が★★以上なので注目
  → 1番は人気馬だけどAI的には妙味なし（エッジがマイナス）
  → 12番は★1つ、余裕があれば押さえ
```

### 注意事項

```
⚠️ 重要: 予測はあくまで「統計的な傾向」です
  - 100%当たることはありません
  - 回収率200%は長期間の平均であり、毎回勝てるわけではありません
  - 1日単位では負けることも普通にあります
  - 馬券購入は必ず余裕のある資金で、自己責任でお願いします
```

## 6. 別のPCで使う方法

### 方法A: USBメモリで持ち運ぶ（簡単）

#### 元のPCでやること

1. プロジェクトフォルダをまるごとUSBメモリにコピー
2. `exports` フォルダにある `keiba_export.tar.gz` も忘れずコピー

#### 別のPCでやること

1. USBメモリからデスクトップなど好きな場所にフォルダをコピー
2. Pythonがなければインストール（[ステップ1](#ステップ1-pythonをインストール)を参照）
3. `setup.bat` をダブルクリック
4. セットアップ完了後、以下のコマンドでデータを取り込む:

```
コマンドプロンプトを開いて:

  cd コピーしたフォルダのパス
  .venv\Scripts\activate
  python scripts\import_data.py exports\keiba_export.tar.gz
```

5. `start.bat` をダブルクリックすればアプリが起動します

### 方法B: Dockerを使う（上級者向け）

1. Docker Desktop をインストール（https://www.docker.com/products/docker-desktop/）
2. プロジェクトフォルダで以下を実行:

```
docker compose up --build
```

3. ブラウザで `http://localhost:8501` を開く

## 7. 困ったときは

### 「start.bat をダブルクリックしても何も起きない」

→ Pythonがインストールされていない可能性があります。
  [ステップ1](#ステップ1-pythonをインストール) の手順でPythonをインストールしてください。
  **「Add python.exe to PATH」のチェック** を忘れないでください。

### 「ブラウザが開かない」

→ 黒い画面に `Local URL: http://localhost:8501` と表示されていれば正常です。
  ブラウザを手動で開いて、アドレス欄に `http://localhost:8501` と入力してください。

### 「予測を実行したが何も表示されない」

→ その日のレースデータがDBに入っていない可能性があります。
  「🔄 データ更新」画面でその月のデータを取得してください。

### 「データ更新が途中で止まる」

→ インターネット接続を確認してください。
  一部失敗しても問題ありません。再度同じ月で「取得」を押せば、
  未取得分だけ追加されます。

### 「予測の計算に時間がかかる」

→ 初回は特徴量の構築に2〜3分かかります。これは正常です。
  2回目以降はキャッシュが効くため、数秒で結果が出ます。
  ブラウザのページを更新するとキャッシュがリセットされます。

### 「モデルが見つかりませんとエラーが出る」

→ `models` フォルダに `.pkl` ファイルがあるか確認してください。
  ない場合は、データを取り込んだ後に以下のコマンドで学習を実行してください:

```
  .venv\Scripts\activate
  python run_train_v3.py
```

### 「精度を上げたい」

→ 以下を試してください:
  1. より多くの年のデータを取得する（`config/settings.py` の `SCRAPE_YEAR_START` を変更）
  2. 定期的にモデルを再学習する（最新データで `python run_train_v3.py` を実行）
  3. バックテスト画面でエッジ閾値を調整し、最適なパラメータを探す

## 用語集

| 用語 | 意味 |
|------|------|
| 複勝 | 3着以内に入れば当たりの馬券 |
| オッズ | 当たったときの配当倍率。10倍なら100円が1000円に |
| 枠番 | スタート位置のグループ番号（1〜8） |
| 馬番 | 個々の馬に割り当てられた番号 |
| 上がり3F | ゴール前600mのタイム。末脚の速さを示す |
| 斤量 | 騎手の体重を含めた馬が背負う重さ |
| 馬場状態 | 芝やダートのコンディション（良→稍重→重→不良） |
| バイアス | 馬場の偏り。内枠有利・外枠有利など |
| 脚質 | 馬の走り方のタイプ（逃げ・先行・差し・追込） |
| スピード指数 | タイムを条件で補正した能力値 |
| 期待値 | 賭けた場合の平均的なリターン |
| エッジ | AIの評価と市場の評価の差 |
| バックテスト | 過去データで「もし賭けていたら」のシミュレーション |
| 回収率 | 賭けた金額に対して戻ってきた金額の割合。100%超で黒字 |
