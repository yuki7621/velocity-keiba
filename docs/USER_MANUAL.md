# velocity-keiba 取り扱い説明書

## はじめに

このアプリは、競馬のレースデータをAIが分析して
**「この馬は人気以上に走りそう」** という馬を見つけてくれるツールです。

難しい操作は必要ありません。基本的に **ボタンを押すだけ** で使えます。

## 1. 最初のセットアップ（初回のみ）

> この作業は最初の1回だけ必要です。2回目以降は不要です。

### 必要なもの

- Windows 10 または 11 のパソコン
- インターネット接続

### 手順

#### ステップ1: Pythonをインストール

1. 以下のサイトを開きます
   - https://www.python.org/downloads/
2. **「Download Python 3.xx.x」** の黄色いボタンをクリック
3. ダウンロードしたファイルを開きます
4. **⚠️ 重要**: 画面下の **「Add python.exe to PATH」にチェックを入れる**
5. **「Install Now」** をクリック
6. 「Setup was successful」と出たら完了

#### ステップ2: アプリをセットアップ

1. このプロジェクトのフォルダを開きます
2. **`setup.bat`** をダブルクリック
3. 黒い画面（コマンドプロンプト）が開いて自動でセットアップが進みます
4. 「セットアップ完了！」と表示されたら完了
5. 何かキーを押して画面を閉じます

```
⏱️ かかる時間: 約3〜5分
```

#### ステップ3: データを取り込む（受け取ったデータがある場合）

誰かから `keiba_export.tar.gz` ファイルを受け取った場合:

1. そのファイルをプロジェクトの `exports` フォルダに入れます
2. `start.bat` をダブルクリックしてアプリを起動
3. 左メニューの **「🔄 データ更新」** は不要です（データは読み込み済み）

データファイルがない場合は、アプリ起動後に自分でデータを取得できます（後述）。

## 2. アプリの起動方法

### 起動する

1. プロジェクトフォルダにある **`start.bat`** をダブルクリック
2. 黒い画面が開いて数秒待つと、自動でブラウザが開きます
3. ブラウザに競馬予想AIの画面が表示されます

```
✅ 画面が出れば成功です！
```

> ブラウザが自動で開かない場合は、手動でブラウザを開いて
> アドレス欄に `http://localhost:8501` と入力してください。

### 終了する

1. 黒い画面（コマンドプロンプト）を閉じるだけでOK
2. ブラウザのタブも閉じてください

## 3. 毎週の使い方（土曜→日曜の流れ）

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│   【土曜の夜】                                             │
│    アプリを起動 → 「データ更新」で土曜のレース結果を取得       │
│    （日付指定タブで今日を選んでボタンを押すだけ）              │
│                                                          │
│          ↓                                               │
│                                                          │
│   【日曜の朝（早朝）】                                      │
│    「予測」で日曜のレースを予測                               │
│    → ★マークの馬をチェック！                                │
│    → 気になるレースは「買い目推奨」で最適な買い方を確認         │
│                                                          │
│          ↓                                               │
│                                                          │
│   【日曜の発走30分前】                                      │
│    馬体重が公開されたら「🔄 このレースを再予測」で更新          │
│    → 買い目推奨でオッズを再取得して最終判断                    │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 土曜の夜にやること（5分で完了）

1. `start.bat` でアプリを起動
2. 左メニューから **「🔄 データ更新」** を選ぶ
3. **「日付指定で取得（推奨）」** タブで今日の日付を選ぶ
4. **「レース結果を取得」** ボタンを押す
5. 進行状況が表示されるので、完了まで待つ
6. 「完了！」と表示されたらOK

```
⏱️ かかる時間: 5〜15分
💡 取得中もPCは普通に使えます
💡 同じ日付を複数回実行しても大丈夫（重複しません）
```

### 日曜の朝にやること（3分で完了）

1. `start.bat` でアプリを起動
2. 左メニューから **「📊 予測」** を選ぶ
3. 「🔮 レース前予測（出馬表から）」モードを選ぶ
4. 「予測する日付」で **今日の日付** を選ぶ
5. 「モデル」は **「lightgbm_v6」**（推奨・血統特徴量あり）のままでOK
6. **「🔮 出馬表を取得して予測」** ボタンを押す
7. しばらく待つと結果が表示されます

```
⏱️ 出馬表の取得＋予測で2〜5分かかります
💡 「📂 全て展開」「📁 全て畳む」で一括切替できます
💡 馬体重が公開されたら「🔄 このレースを再予測」で更新できます
```

### 買い目を決める（発走前）

1. 予測結果が表示された状態で **「💰 買い目推奨」** に切り替え
2. 対象レースを選択
3. 予算を入力（例: 1000円）
4. **「🔄 オッズ取得 & 買い目計算」** を押す
5. EV（期待値）の高い買い目が予算内で提案されます

## 4. 各画面の使い方

### 📊 予測（メイン画面）

**何ができる？**: 指定した日のレースで「買い」の馬を見つけます

**2つのモード**:
- **🔮 レース前予測（出馬表から）**: 当日の予測に使う（メイン）
- **📁 DB内データで予測**: 過去レースの検証用

#### レース前予測の操作手順
1. 予測する日付を選ぶ（デフォルトは今日）
2. 競馬場を絞りたければ選択（空欄で全会場）
3. **「🔮 出馬表を取得して予測」** を押す
4. 結果が表示されたら:
   - **「📂 全て展開」「📁 全て畳む」** で一括切替
   - 各レースの **「🔄 このレースを再予測」** で馬体重公開後に更新

**表示される情報**:
- **前日の馬場傾向**: 芝/ダート別のバイアス
- **注目馬ランキング**: AI確率Top20
- **レース別予測**: 全レースの詳細（上位馬のコメント付き）

### 📝 note記事作成

**何ができる？**: 予測結果から、会場別にまとめた配布用記事(Markdown形式)を自動生成します。noteプラットフォームにそのまま貼り付けできます。

**操作手順**:
1. 先に「📊 予測」ページで予測を実行しておく
2. 「📝 note記事作成」に切り替え
3. 掲載する会場を選ぶ（デフォルト全会場）
4. 全レース掲載 or 重賞・特別戦のみを選ぶ
5. 相手馬（○▲△）の掲載ON/OFFを選ぶ
6. 冒頭あいさつ・末尾注意書きをお好みで編集
7. 生成されたMarkdownをコピー or ダウンロード

**記事の構成例**:
```
# 【2026年4月27日（日）】競馬AI予想 東京・京都

## 🏇 東京
### 11R 天皇賞（芝3200m）
◎ 5番 ○○○○○ (AI確率 42.3%)

推奨理由:
- 近5走の複勝率 60% と安定した成績
- 同距離帯で複勝率 67% と適性◎
- 父キタサンブラック産駒は芝で複勝率 35%（v6血統特徴量）
- ...

○ 3番 △△△△ (AI確率 30.1%)
▲ 7番 □□□□ (AI確率 25.8%)
```

**マークの意味**:
| マーク | 意味 |
|--------|------|
| ◎ | 本命（1番手評価・推奨理由付き） |
| ○ | 対抗（2番手評価） |
| ▲ | 単穴（3番手評価） |
| △ | 連下（4-5番手評価） |

### 💰 買い目推奨

**何ができる？**: 予測結果と現在オッズから、全7券種の最適な買い目を提案します

**操作手順**:
1. 先に「予測」ページで予測を実行しておく
2. 「💰 買い目推奨」に切り替え
3. 対象レースを選択
4. 予算・EV閾値・配分方式を設定
5. **「🔄 オッズ取得 & 買い目計算」** を押す

**配分方式の選び方**:
| 方式 | おすすめの人 |
|------|------------|
| EV順に均等買い | 初心者・安全に運用したい人 |
| Half Kelly | モデルの精度に自信が出てきたら（推奨） |
| Full Kelly | 上級者・攻めたい人（変動大） |
| EVに比例配分 | バランスを取りたい人 |

### 📈 馬場傾向

**何ができる？**: 特定の日の馬場の傾向を **芝/ダート別** に詳しく分析します

**操作手順**:
1. 競馬場を選ぶ
2. 日付を選ぶ
3. 「分析実行」を押す

**表示される情報**（芝/ダートそれぞれタブ表示）:
- **枠順バイアス**: 内枠と外枠、どちらが有利だったか
- **脚質バイアス**: 逃げ先行と差し追込、どちらが有利だったか
- **馬場スピード**: 時計が速い馬場か、タフな馬場か
- **上がり傾向**: 末脚勝負になりやすいかどうか

**見方のコツ**:
```
例: 芝タブで「内枠有利 ◎」と出たら...
  → 日曜の芝レースも同じ傾向が続く可能性が高い
  → 芝レースで内枠の馬を重視する判断材料になる

例: ダートタブで「先行有利 ◎」と出たら...
  → ダートレースでは前に行ける馬を重視する
```

### 🔄 データ更新

**何ができる？**: netkeiba.com から最新のレース結果を取得します

**2つのタブ**:
- **📌 日付指定で取得（推奨）**: 日付を選んで当日〜前日のデータを素早く取得
- **📅 月単位で取得**: 従来方式。年月を選んでまとめて取得

**推奨の使い方**:
1. 「日付指定で取得」タブで今日の日付を選ぶ
2. **「レース結果を取得」** を押す
3. 既にDBにあるレースは自動スキップされるので、何度実行しても安全

**注意点**:
- 取得中はブラウザを閉じないでください
- インターネット接続が必要です
- netkeiba.com への負荷を避けるため、1レースごとに2秒間隔を空けています

### 🧪 バックテスト

**何ができる？**: 「もし過去にこの戦略で賭けていたら回収率はどうだったか？」をシミュレーションします

**操作手順**:
1. モデルを選ぶ（通常は「lightgbm_v6」）
2. 戦略を選ぶ:
   - **バリューベット（エッジ）**: おすすめ。AIと市場の確率差で判断
   - **期待値（EV）**: 期待リターンで判断
3. オッズ範囲を設定（初期値のままでOK）
4. 閾値を選ぶ（複数選択可）
5. 「バックテスト実行」を押す

**結果の見方**:
- **回収率100%超**: この戦略なら利益が出ていた
- **回収率100%未満**: この戦略では損をしていた
- **グラフ**: 閾値を上げると賭け数は減るが回収率は上がる傾向

## 5. 予測結果の見方

### ★マークの意味

| マーク | 意味 | おすすめ度 |
|--------|------|-----------|
| ★★★ | AIが「かなりお買い得」と判断 | 最もおすすめ |
| ★★ | AIが「お買い得」と判断 | おすすめ |
| ★ | AIが「やや有利」と判断 | 注目 |
| （なし） | AIが「妥当な評価」と判断 | 見送り |

### 数値の見方

#### AI確率
AIが計算した「この馬が3着以内に入る確率」です。
```
  高い例: 65.2% → AIはかなり来ると予測
  低い例: 8.3%  → AIはあまり来ないと予測
```

#### オッズ
馬券売り場で表示される単勝オッズです。
```
  低い例: 2.5倍 → 人気馬（みんなが買っている）
  高い例: 45.0倍 → 人気のない馬（あまり買われていない）
```

#### 期待値
「この馬に100円賭けたら平均いくら返ってくるか」の目安です。
```
  1.00以上 → 期待リターンがプラス（買う価値あり）
  1.00未満 → 期待リターンがマイナス

  例: 期待値 1.75 → 100円賭けると平均175円返ってくる計算
```

#### エッジ
AIの評価と市場の評価の差です。プラスが大きいほどお買い得。
```
  +0.20 → AIは市場より20%も高く評価（かなりお買い得）
  +0.10 → AIは市場より10%高く評価（お買い得）
  +0.02 → ほぼ市場通りの評価（妙味なし）
  -0.05 → 市場のほうが高く評価（人気しすぎ）
```

### 具体的な使い方の例

```
日曜の朝、予測を実行して以下の結果が出たとします:

  中山11R 芝2000m
  馬番  馬名          AI確率  オッズ  期待値  エッジ   推奨
   3   ○○○○○○     38.1%   15.3   1.75   +0.184  ★★★
   7   △△△△△      29.5%   28.4   2.52   +0.144  ★★
   1   □□□□□□     52.1%    2.8   0.44   -0.021
  12   ▽▽▽▽▽      11.3%   42.0   1.43   +0.042  ★

  → 3番と7番が★★以上なので注目
  → 1番は人気馬だけどAI的には妙味なし（エッジがマイナス）
  → 12番は★1つ、余裕があれば押さえ
```

### 注意事項

```
⚠️ 重要: 予測はあくまで「統計的な傾向」です
  - 100%当たることはありません
  - 回収率200%は長期間の平均であり、毎回勝てるわけではありません
  - 1日単位では負けることも普通にあります
  - 馬券購入は必ず余裕のある資金で、自己責任でお願いします
```

## 6. 買い目推奨の使い方

### 基本的な流れ

1. **「予測」ページでレース予測を実行**（必須・先にやる）
2. **「買い目推奨」ページに切り替え**
3. レース選択 → 予算入力 → 計算実行

### 設定項目

| 項目 | 説明 | おすすめ値 |
|------|------|-----------|
| 予算 | 1レースに使う金額 | 1,000〜3,000円 |
| EV閾値 | 最低期待値（1.0 = 損益分岐点） | 1.10 |
| 配分方式 | 購入金額の配分方法 | EV順に均等買い（最初は） |
| 対象券種 | 計算する券種 | 単勝・複勝・馬連・ワイドが手堅い |

### 結果の見方

```
💎 推奨買い目（予算内）
  → 予算内でEVが高い順に選ばれた買い目

📊 全候補一覧
  → 全ての買い目候補（EV順）
  → 🔥 EV≥1.3（かなり美味しい）
  → ⭐ EV≥1.1（プラス期待）
  → 🟢 EV≥1.0（損益分岐点以上）
  → ⚪ EV<1.0（マイナス期待）
```

### 配分方式のステップアップ

```
【最初の3ヶ月】 EV順に均等買い で運用 → データを貯める
     ↓
【3ヶ月後】 モデル診断の較正グラフが対角線に近ければ
     → Half Kelly に切替
     ↓
【半年後】 安定していれば → Full Kelly も検討
```

## 7. note記事作成の使い方

### 基本的な流れ

1. **「📊 予測」ページで予測を実行**（必須・先にやる）
2. **「📝 note記事作成」ページに切り替え**
3. 設定を確認して、生成されたMarkdownをコピー or ダウンロード

### おすすめの使い方（週次配信を想定）

```
土曜の夜   → データ更新 → 予測実行
          ↓
日曜の朝   → 馬体重発表後に「🔄 このレースを再予測」
          ↓
          → 「📝 note記事作成」でMarkdown生成
          ↓
          → noteに貼り付けて公開
```

### 推奨理由の自動生成

本命馬（◎）に対して、AIが特徴量から5つまで選んで理由を付けます:
- 近5走の複勝率が高い
- 同距離帯で好走歴あり
- 得意コース・得意馬場
- 鞍上騎手が好調
- 血統的に適性あり（v6モデルのみ）
- 調子上昇トレンド
- 2番手馬を大きく引き離した評価

### 出力のカスタマイズ

- **全レース or 特別戦のみ**: チェックボックスで切替
- **会場絞り込み**: 「東京だけ」「東京・京都」など選択可
- **相手馬の有無**: ○▲△を載せるかどうか
- **冒頭あいさつ・末尾注意書き**: 自由編集可（固定テキストを再利用したい場合に便利）

## 8. モデル診断の使い方

### 週次チェック（毎週5分）

1. 「🔬 モデル診断」を開く
2. 「📅 週次チェック」タブを確認
3. アラートが出ていなければOK

**アラートが出たら**:
- ROI低下: 一時的な変動の可能性。2〜3週間続くなら改善を検討
- 的中率低下: 馬場状態の変化や季節要因の可能性あり

### 期間診断（月1回）

1. 「🔬 期間診断」タブを開く
2. 期間を選択（「3ヶ月」がおすすめ）
3. 「診断実行」を押す
4. 結果を確認
5. **「📋 Claude改善依頼レポート」ボタン** でレポートをダウンロード
6. ダウンロードしたレポートをClaudeに貼り付けて改善を依頼

```
💡 レポートには以下が含まれます:
  - モデルの現状の数値
  - 弱点（苦手な条件・距離・馬場など）

  - 改善提案
  
  → Claudeに「このレポートを元にモデルを改善してください」と
     依頼すれば、具体的なコード修正を提案してくれます
```

## 9. 別のPCで使う方法

> **注意**: 別のPCにコピーする際、`.venv` フォルダは**コピーしないでください**。
> `.venv` はPC固有のパスが含まれるため、コピー先では動きません。
> `setup.bat` を実行すれば自動で `.venv` が作り直されます。

### 方法A: USBメモリで持ち運ぶ（簡単）

#### 元のPCでやること

1. プロジェクトフォルダをまるごとUSBメモリにコピー
2. `exports` フォルダにある `keiba_export.tar.gz` も忘れずコピー

#### 別のPCでやること

1. USBメモリからデスクトップなど好きな場所にフォルダをコピー
2. Pythonがなければインストール（[ステップ1](#ステップ1-pythonをインストール)を参照）
3. `setup.bat` をダブルクリック
4. セットアップ完了後、以下のコマンドでデータを取り込む:

```
コマンドプロンプトを開いて:

  cd コピーしたフォルダのパス
  .venv\Scripts\activate
  python scripts\import_data.py exports\keiba_export.tar.gz
```

5. `start.bat` をダブルクリックすればアプリが起動します

### 方法B: Dockerを使う（上級者向け）

1. Docker Desktop をインストール（https://www.docker.com/products/docker-desktop/）
2. プロジェクトフォルダで以下を実行:

```
docker compose up --build
```

3. ブラウザで `http://localhost:8501` を開く

## 10. 困ったときは

### 「start.bat をダブルクリックしても何も起きない」

→ Pythonがインストールされていない可能性があります。
  [ステップ1](#ステップ1-pythonをインストール) の手順でPythonをインストールしてください。
  **「Add python.exe to PATH」のチェック** を忘れないでください。

### 「ブラウザが開かない」

→ 黒い画面に `Local URL: http://localhost:8501` と表示されていれば正常です。
  ブラウザを手動で開いて、アドレス欄に `http://localhost:8501` と入力してください。

### 「予測を実行したが何も表示されない」

→ その日のレースデータがDBに入っていない可能性があります。
  「🔄 データ更新」画面でその月のデータを取得してください。

### 「データ更新が途中で止まる」

→ インターネット接続を確認してください。
  一部失敗しても問題ありません。再度同じ月で「取得」を押せば、
  未取得分だけ追加されます。

### 「予測の計算に時間がかかる」

→ 初回は特徴量の構築に2〜3分かかります。これは正常です。
  2回目以降はキャッシュが効くため、数秒で結果が出ます。
  ブラウザのページを更新するとキャッシュがリセットされます。

### 「モデルが見つかりませんとエラーが出る」

→ `models` フォルダに `.pkl` ファイルがあるか確認してください。
  ない場合は、データを取り込んだ後に以下のコマンドで学習を実行してください:

```
  .venv\Scripts\activate
  python run_train_v6.py
```

### 「精度を上げたい」

→ 以下を試してください:
  1. より多くの年のデータを取得する（`config/settings.py` の `SCRAPE_YEAR_START` を変更）
  2. 定期的にモデルを再学習する（最新データで `python run_train_v6.py` を実行）
  3. バックテスト画面でエッジ閾値を調整し、最適なパラメータを探す
  4. 血統データが未取得なら `python run_scrape_pedigree.py` を実行（約10時間）し v6 を再学習
